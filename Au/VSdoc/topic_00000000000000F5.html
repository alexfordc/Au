AO role, like "LINK". Or path, like "ROLE/ROLE/ROLE". See Role. Can be used standard roles (see AccROLE) and custom roles (like "div" in Firefox). This parameter is string. If you want to use AccROLE: nameof(AccROLE.CHECKBOX). Case-sensitive, not wildcard. Use null to match any role. Cannot be "". Role or path can have a prefix:

Cannot use prefix when prop contains "id" or "class". Also with flag UIA. Also when searching in Acc (Find(string,string,string,AFFlags,Func<Acc, bool>, int)). Can be used path consisting of roles separated by "/". Examples:

name
Type: string

AO name (Name). String format: <conceptualLink target="0248143b-a0dd-4fa1-84f9-76831db6714a">wildcard expression</conceptualLink>. null means 'any'. "" means 'empty or unavailable'.

prop
Type: string

Other AO properties and search settings. Format: one or more "name=value", separated with "\0" or "\0 ", like "description=xxx\0 @href=yyy". Names must match case. Values of string properties are wildcard expressions.

flags
Type: AFFlags

also
Type: Func<Acc, bool>

Lambda etc callback function to call for each matching AO. Let it return true if this is the wanted AO. Example, the AO must contain point x y: o => o.GetRect(out var r, o.WndTopLevel) && r.Contains(266, 33)

skip
Type: int

0-based index of matching AO. For example, if 1, the function skips the first matching AO and returns the second.

controls
Type: ChildFinder

Properties of child controls where to search. This is an alternative for class/id in prop. Allows to specify more control properties. Works better/faster when the control is of a different process or thread than the parent window; else slightly slower.

Remarks
This function walks the tree of accessible objects of the window, control or web page, until it finds a matching AO. Uses Find(Wnd). You can use it directly (see example). In wildcard expressions supports PCRE regular expressions (prefix "**r ") but not .NET regular expressions (prefix "**R "). They are similar. To find web page AOs usually it's better to use Wait(double,Wnd,string,string,string,AFFlags,Func<Acc, bool>, int) instead, it's more reliable. More info in class help.
Example
Find link "Example" in web page, and click. Throw NotFoundException if not found.
Copy 
 var w = +Wnd.Find("* Chrome");
 var a = +Acc.Find(w, "web:LINK", "Example");
 a.DoAction();
Try to find link "Example" in web page. Return if not found.
Copy 
 var w = +Wnd.Find("* Chrome");
 var a = Acc.Find(w, "web:LINK", "Example");
 if(a == null) { Print("not found"); return; }
 a.DoAction();
Use a Finder.
Copy 
 var w = +Wnd.Find("* Chrome");
 var f = new Acc.Finder("BUTTON", "Example");
 if(!f.Find(w)) { Print("not found"); return; }
 Acc a = f.Result;
 a.DoAction();
