Wnd Structure

Example See Also

A variable of Wnd type represents a window or control. It is a window handle, also known as HWND. Wnd functions can be used with windows and controls of any process/thread. Wnd functions also can be used with .NET form/control and WPF window class variables, like Wnd w=(Wnd)form; w.Method(...); or ((Wnd)form).Method(...);.

ZorderTop() Places this window or control at the top of the Z order. WButton Like Wnd, but has only button, check box and radio button functions - Click, Check etc.
Top
Remarks
There are two main types of windows - top-level windows and controls. Controls are child windows of top-level windows. More functions are in the nested classes - Misc, Desktop etc. They are used mostly in programming, rarely in automation scripts. What happens when a Wnd function fails:
Many functions fail if the window's process has a higher UAC integrity level (aministrator, uiAccess) than this process, unless this process has uiAccess level. Especially the functions that change window properties. Some functions that still work: Activate, ActivateLL, ShowMinimized, ShowNotMinimized, ShowNotMinMax, Close. The Wnd type can be used with native Windows API functions without casting. Use Wnd for the parameter type in the declaration, like [DllImport(...)] static extern bool NativeFunction(Wnd hWnd, ...). See also: MSDN article <msdn>Window Features</msdn>.
Example
Copy 
 Wnd w = Wnd.Find("* - Notepad");
 if(w.Is0) { Print("window not found"); return; }
 w.Activate();
 Wnd c = w.Child(className: "Edit");
 Print(c.Name);
