Parameters

text1
Type: string

Main instruction. Bigger font.

text2
Type: string

Text below main instruction.

buttons
Type: string

Button ids and labels, like "1 OK|2 Cancel|5 Save|4 Don't Save". Missing ids are auto-generated, for example "OK|Cancel|100 Custom1|Custom2" is the same as "1 OK|2 Cancel|100 Custom1|101 Custom2". The first in the list button is default, ie responds to the Enter key. For example, "2 No|1 Yes" adds Yes and No buttons and makes No default. Trims newlines around ids and labels. For example, "\r\n1 One\r\n|\r\n2\r\nTwo\r\n\r\n" is the same as "1 One|2 Two". To create keyboard shortcuts, use & character in custom button labels. Use && for literal &. Example: "1 &Tuesday[]2 T&hursday[]3 Saturday && Sunday". There are 6 common buttons: OK, Yes, No, Retry, Cancel, Close. Buttons that have other labels are custom buttons. How common buttons are different: 1. DFlags.CommandLinks does not change their style. 2. They have keyboard shortcuts that cannot be changed. Inserting & in a label makes it a custom button. 3. Button Cancel can be selected with the Esc key. It also adds X (Close) button in title bar, which selects Cancel. 4. Always displayed in standard order (eg Yes No, never No Yes). But you can for example use "2 No|1 Yes" to set default button = No. 5. The displayed button label is localized, ie different when the Windows UI language is not English. If omitted, null or "", the dialog will have OK button, id 1. You can use CommandLinks in flags to change the style of custom buttons. See also: SetButtons.

Remarks
Calls ShowEx. Tip: For optional parameters use named arguments. Example: AuDialog.Show("Text.", icon: DIcon.Info) .

 if(AuDialog.Show("Show another example?", null, "1 OK|2 Cancel", DIcon.Info) != 1) return;
 Print("OK");
 
 switch(AuDialog.Show("Save changes?", "More info.", "1 Save|2 Don't Save|Cancel")) {
 case 1: Print("save"); break;
 case 2: Print("don't"); break;
 default: Print("cancel"); break;
 }
