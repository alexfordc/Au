Wnd Methods

Wnd Structure  See Also

The following tables list the members exposed by the Wnd type.

Methods
Name Description
Activate() Activates this window (brings to the foreground). Also unhides, restores minimized etc, to ensure that the window is ready to receive sent keys, mouse clicks ect.
ActivateLL() Low-level version of Activate. Just calls EnableActivate, API <msdn>SetForegroundWindow</msdn> and makes sure that it actually worked, but does not check whether it activated exactly this window. No exceptions, does not unhide, does not restore minimized, does not check is it a top-level window or control, etc. Returns false if fails.
AllChildren(bool, bool, bool) Gets child controls. Returns array containing 0 or more control handles as Wnd.
ButtonClick(int, bool) Finds a child button by id and sends a "click" message. Does not use the mouse. Calls Click(bool).
ButtonClick(string, string, bool) Finds a child button by name and sends a "click" message. Does not use the mouse. Calls Click(bool).
Child(string, string, WCFlags, Func<WndBoolean>, int) Finds a child control and returns its handle as Wnd. Returns default(Wnd) if not found. See also: Is0, OrThrow(Wnd), operator+(Wnd).
ChildAll(string, string, WCFlags, Func<WndBoolean>) Finds all matching child controls. Returns List containing 0 or more control handles as Wnd. Everything except the return type is the same as with Child.
ChildFast(string, string, Wnd) Finds a direct child control and returns its handle as Wnd. Returns default(Wnd) if not found. See also: Is0, OrThrow(Wnd), operator+(Wnd). Calls API <msdn>FindWindowEx</msdn>. Faster than Child, which uses API <msdn>EnumChildWindows</msdn>. Can be used only when you know full name and/or class name. Finds hidden controls too.
ChildFromXY(Coord, Coord, bool, bool) Gets child control from point. Returns default(Wnd) if the point is not in a child control or not in the client area of this window.
ClassNameIs(string) Returns true if the class name of this window matches className. Else returns false. Also returns false when fails (probably window closed or 0 handle). Supports GetError.
ClassNameIs(params string[]) If the class name of this window matches one of strings in classNames, returns 1-based index of the string. Else returns 0. Also returns 0 if fails to get class name (probably window closed or 0 handle). Supports GetError.
Close(bool, bool) Closes the window. Returns true if successfuly closed or if it was already closed (the handle is 0 or invalid) or if noWait==true.
ContainsScreenXY(Coord, Coord, CoordOptions) Returns true if this window (its rectangle) contains the specified point in primary screen coordinates.
ContainsWindowXY(Coord, Coord) This overload calls ContainsWindowXY(WndWindow, x, y).
ContainsWindowXY(Wnd, Coord, Coord) Returns true if this control (its rectangle) contains the specified point in parent window.
Enable(bool) Enables or disables the window. Calls API <msdn>EnableWindow</msdn>.
EnsureInScreen(object, bool) Moves this window if need, to ensure that entire window is in screen.
Equals(Wnd) Returns true if w == this.
Equals(Nullable<Wnd>) Returns true if w != null and w.Value == this.
Equals(object) Returns true if obj is this Wnd.
Find(string, string, string, WFFlags, Func<WndBoolean>, object) Finds a top-level window and returns its handle as Wnd. Returns default(Wnd) if not found. See also: Is0, OrThrow(Wnd), operator+(Wnd). See examples.
FindAll(string, string, string, WFFlags, Func<WndBoolean>, object) Finds all matching windows. Returns array containing 0 or more window handles as Wnd. Everything except the return type is the same as with Find.
FindFast(string, string, Wnd) Finds a top-level window and returns its handle as Wnd. Returns default(Wnd) if not found. To check it you can use Is0 or OrThrow(Wnd). Calls API <msdn>FindWindowEx</msdn>. Faster than Find, which uses API <msdn>EnumWindows</msdn>. Can be used only when you know full name and/or class name. Finds hidden windows too.
FindOrRun(string, string, string, WFFlags, Func<WndBoolean>, object, Func<Int32>, double, bool) Finds a top-level window. If found, activates (optionally), else calls callback function that should open the window (eg call Run) and waits for the window. Returns window handle as Wnd. Returns default(Wnd) if not found (if runWaitTimeoutS is negative; else exception). The first 5 parameters are the same as Find.
Focus() Sets the keyboard input focus to this control or window. Also activetes its top-level parent window (see Activate). Can belong to any process/thread. With controls of this thread you can use the more lightweight function FocusLocal.
FocusLocal() Calls API <msdn>SetFocus</msdn>, which sets the keyboard input focus to this control or window, which must be of this thread. Fails if it belongs to another thread or is invalid or disabled. Returns false if fails. Supports GetError.
FromMouse(WXYFlags) Gets visible top-level window or control from mouse cursor position. More info: FromXY(Coord,Coord,WXYFlags,CoordOptions).
FromXY(Coord, Coord, WXYFlags, CoordOptions) Gets visible top-level window or control from point. By default the coordinates are relative to the primary screen.
FromXY(Point, WXYFlags) Gets visible top-level window or control from point. More info: FromXY(Coord,Coord,WXYFlags,CoordOptions).
GetClientRect(out RECT) Gets client area rectangle. The same as the ClientRect property. The same as GetClientSize, just the parameter type is different.
GetClientSize(out Size) Gets client area width and height. The same as the ClientSize property. The same as GetClientRect, just the parameter type is different.
GetHashCode() Returns the hash code for this instance.
GetRect(out RECT) Gets rectangle (position and size) in screen coordinates. The same as the Rect property.
GetRectInClientOf(Wnd, out RECT) Gets rectangle of this window (usually control) relative to the client area of another window (usually the parent).
GetRectNotMinMax(out RECT) Gets rectangle of normal (restored) window even if currently it is minimized or maximized.
GetSize(out Size) Gets width and height. The same as the Size property.
GetText(Nullable<Boolean>, bool) Gets window/control name or control text. Returns "" if it is empty. Returns null if fails, eg if the window is closed. Supports GetError. This is a low-level function. You can instead use Name and ControlText.
GetThreadProcessId(out int) Calls API <msdn>GetWindowThreadProcessId</msdn>. Returns thread id and also gets process id. Returns 0 if fails. Supports GetError. <note>It is native thread id, not Thread.ManagedThreadId.</note>
GetWindowAndClientRectInScreen(out RECT, out RECT) Gets window rectangle and client area rectangle, both in screen coordinates.
GetWindowLong(int) Calls API GetWindowLong if this process is 32-bit, GetWindowLongPtr if 64-bit.
HasAcc(Finder) Returns true if this window contains the specified accessible object. Calls ChildFinder).
HasChild(ChildFinder) Returns true if this window contains the specified control. Calls Find.
HasChild(string, string, WCFlags, Func<WndBoolean>, int) Returns true if this window contains the specified control. Calls Child. <note type="note"> Using this function many times with same parameters is inefficient. Instead create new ChildFinder and call Find or HasChild(ChildFinder). See example. </note>
HasExStyle(UInt32) Returns true if the window has all specified extended style flags (see ExStyle).
HasStyle(UInt32) Returns true if the window has all specified style flags (see Style).
IsChildOf(Wnd) Returns true if this is a direct or indirect child (descendant) of window w. Calls API <msdn>IsChild</msdn>.
IsSpecHwnd(Wnd) Returns true if w contains a non-zero special handle value (see SpecHwnd). Note that HWND_TOP is 0.
Kid(int, WCFlags) Finds a child control by its id and returns its handle as Wnd. Returns default(Wnd) if not found. See also: Is0, OrThrow(Wnd), operator+(Wnd).
MapClientToClientOf(Wnd, ref Point) Converts coordinates relative to the client area of this window to coordinates relative to the client area of window w.
MapClientToClientOf(Wnd, ref RECT) Converts coordinates relative to the client area of this window to coordinates relative to the client area of window w.
MapClientToScreen(ref Point) Converts coordinates relative to the client area of this window to coordinates relative to the screen.
MapClientToScreen(ref RECT) Converts coordinates relative to the client area of this window to coordinates relative to the screen.
MapClientToWindow(ref Point) Converts coordinates relative to the client area of this window to coordinates relative to the top-left corner of this window.
MapClientToWindow(ref RECT) Converts coordinates relative to the client area of this window to coordinates relative to the top-left corner of this window.
MapScreenToClient(ref Point) Converts coordinates relative to the screen to coordinates relative to the client area of this window.
MapScreenToClient(ref RECT) Converts coordinates relative to the screen to coordinates relative to the client area of this window.
MapWindowToClient(ref Point) Converts coordinates relative to the top-left corner of this window to coordinates relative to the client area of this window.
MapWindowToClient(ref RECT) Converts coordinates relative to the top-left corner of this window to coordinates relative to the client area of this window.
MapWindowToScreen(ref Point) Converts coordinates relative to the top-left corner of this window to screen coordinates.
MapWindowToScreen(ref RECT) Converts coordinates relative to the top-left corner of this window to screen coordinates.
MenuClick(int, bool) Posts a "menu item clicked" notification (<msdn>WM_COMMAND</msdn>) as if that menu item was clicked. Does not use the mouse.
Move(Coord, Coord, Coord, Coord, CoordOptions) Moves and/or resizes. With windows of current thread usually it's better to use MoveLL(int,int,int,int,uint).
Move(Coord, Coord, CoordOptions) Moves. Calls Move(x, y, null, null, workArea). With windows of current thread usually it's better to use MoveLL(int,int).
MoveInScreen(Coord, Coord, object, bool, bool) Moves this window to coordinates x y in specified screen, and ensures that entire window is in screen.
MoveLL(int, int) Moves.
MoveLL(int, int, int, int, UInt32) Moves and resizes.
MoveToScreenCenter(object) Moves this window to the center of the screen. Calls ShowNotMinMax(true) and MoveInScreen(null, null, screen, true, true).
Post(UInt32, LPARAM, LPARAM) Calls API <msdn>PostMessage</msdn>. Returns its return value (false if failed). Supports GetError.
Resize(Coord, Coord, CoordOptions) Resizes. Calls Move(null, null, width, height, workArea). With windows of current thread usually it's better to use ResizeLL(int,int).
ResizeLL(int, int) Resizes.
RestorePositionSizeState(string, bool, bool) Restores window position, size and state that is stored in a string created by SavePositionSizeState.
SavePositionSizeState(bool) Gets window position, size and state stored in a string that can be used with RestorePositionSizeState. Returns null if failed. Supports GetError.
Send(UInt32, LPARAM, LPARAM) Calls API <msdn>SendMessage</msdn>.
SendNotify(UInt32, LPARAM, LPARAM) Calls API <msdn>SendNotifyMessage</msdn>. Returns its return value (false if failed). Supports GetError.
SendS(UInt32, LPARAM, char[]) Calls API <msdn>SendMessage</msdn> where lParam is char[].
SendS(UInt32, LPARAM, string) Calls API <msdn>SendMessage</msdn> where lParam is string.
SendTimeout(int, out LPARAM, UInt32, LPARAM, LPARAM, UInt32) Calls API <msdn>SendMessageTimeout</msdn> and gets the result of the message processing. Returns its return value (false if failed). Supports GetError. flags can be Native.SMTO_.
SendTimeout(int, UInt32, LPARAM, LPARAM, UInt32) Calls API <msdn>SendMessageTimeout</msdn>. Returns its return value (false if failed). Supports GetError. flags can be Native.SMTO_.
SendTimeoutS(int, out LPARAM, UInt32, LPARAM, char[], UInt32) Calls API <msdn>SendMessageTimeout</msdn> where lParam is char[]. Returns its return value (false if failed). Supports GetError. flags can be Native.SMTO_.
SendTimeoutS(int, out LPARAM, UInt32, LPARAM, string, UInt32) Calls API <msdn>SendMessageTimeout</msdn> where lParam is string. Returns its return value (false if failed). Supports GetError. flags can be Native.SMTO_.
SetClientSize(Nullable<Int32>, Nullable<Int32>) Calculates and sets window rectangle from the specified client area rectangle. Calls ResizeLL.
SetExStyle(UInt32, SetAddRemove, bool, bool) Changes window extended style.
SetStyle(UInt32, SetAddRemove, bool, bool) Changes window style.
SetText(string) Sets window/control name or control text.
SetTransparency(bool, Nullable<Double>, Nullable<ColorInt>) Sets transparency. On Windows 7 works only with top-level windows, on newer OS also with controls.
SetWindowLong(int, LPARAM) Calls API <msdn>SetWindowLong</msdn> if this process is 32-bit, SetWindowLongPtr if 64-bit.
SetWindowPos(UInt32, int, int, int, int, Wnd) Calls API <msdn>SetWindowPos</msdn>.
Show(bool) Shows (if hidden) or hides this window. Does not activate/deactivate/zorder. With windows of current thread usually it's better to use ShowLL.
ShowLL(bool) Shows (if hidden) or hides this window. Does not activate/deactivate/zorder.
ShowMaximized(bool) If not minimized, minimizes. Also unhides.
ShowMinimized(bool) If not minimized, minimizes. Also unhides.
ShowNotMinMax(bool) If maximized or minimized, makes normal (not min/max). Also unhides.
ShowNotMinimized(bool) If minimized, restores previous non-minimized state (maximized or normal). Also unhides.
ThrowIf0() If Is0, throws WndException.
ThrowIfInvalid() If Is0 or !IsAlive, throws WndException.
ThrowNoNative(string) Throws WndException that uses mainMessage and does not use the last Windows API error. Also the message depends on whether the window handle is 0/invalid.
ThrowUseNative() Throws WndException that uses the last Windows API error (code and message). Also the message depends on whether the window handle is 0/invalid.
ThrowUseNative(int, string) Throws WndException that uses mainMessage and the specified Windows API error code. Also the message depends on whether the window handle is 0/invalid.
ThrowUseNative(string) Throws WndException that uses mainMessage and the last Windows API error (code and message). Also the message depends on whether the window handle is 0/invalid.
ToString() Formats string $"{handle} {ClassName} \"{Name}\" {ProcessName} {Rect}".
WndChild(int) Gets the child control at the specified position in the Z order. Returns default(Wnd) if no children or if index is invalid.
ZorderAbove(Wnd) Places this window before window w in the Z order. Also can make this window topmost or non-topmost, depending on where w is in the Z order. This window and w can be both top-level windows or both controls of same parent. May not work with top-level windows when it would move an inactive window above the active window. If w is default(Wnd), calls ZorderBottom.
ZorderBelow(Wnd) Places this window after window w in the Z order. Also can make this window topmost or non-topmost, depending on where w is in the Z order. This window and w can be both top-level windows or both controls of same parent. May not work with top-level windows when it would move an inactive window above the active window. If w is default(Wnd), calls ZorderTop.
ZorderBottom() Places this window or control at the bottom of the Z order. If the window was topmost, makes it and its owner window non-topmost.
ZorderIsBefore(Wnd) Returns true if this window is above window w in the Z order.
ZorderNoTopmost(bool) Makes this window non-topmost. If this window has an owner window, makes the owner window non-topmost too. This cannot be a control.
ZorderTop() Places this window or control at the top of the Z order. If the window was topmost, it will be at the top of topmost windows, else at the top of non-topmost windows (after topmost windows). Does not activate. In most cases does not work with top-level inactive windows, although returns true; instead use ActivateLL.
ZorderTopmost() Makes this window topmost (always on top of non-topmost windows in the Z order). Does not activate. If this window has an owner window, the owner does not become topmost. This cannot be a control.
