Remarks
Specialized 'wait for' functions are in other classes, for example Wnd.Wait.

All 'wait for' functions have a secondsTimeout parameter. It is the maximal time to wait, seconds. If it is 0, waits infinitely. If >0, after that time interval throws System.TimeoutException. If <0, then stops waiting and returns default value of that type (false, etc).

While waiting, most functions by default don't dispatch Windows messages, events, hooks, timers, COM/RPC, etc. For example, if used in a Form/Control event handler, the form would stop responding. Use another thread, for example async/await/Task, like in the example. Or option Opt.WaitFor.DoEvents.
Examples
Copy
WaitFor.Condition(0, () => Keyb.IsScrollLock);
Print("ScrollLock now is toggled");
Using in a Form/Control event handler.
Copy
var f = new Form();
f.Click += async (unu, sed) =>
  {
