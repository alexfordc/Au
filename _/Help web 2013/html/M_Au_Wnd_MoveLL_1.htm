Remarks
See also Wnd.Move. It is better to use in automation scripts, with windows of any process/thread. It throws exceptions, supports optional/reverse/fractional/workarea coordinates, restores if min/max, does not support SWP flags.
This function is low-level, it just calls API SetWindowPos with flags NOZORDER|NOOWNERZORDER|NOACTIVATE|swpFlagsToAdd. It is better to use in programming, with windows of current thread.
Supports WinError.Code.

For top-level windows use screen coordinates. For controls - direct parent client coordinates.
