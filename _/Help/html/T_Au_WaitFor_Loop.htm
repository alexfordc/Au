MaxPeriod
Maximal period (WaitFor.Loop.Sleep sleep time), milliseconds.

Current period (WaitFor.Loop.Sleep sleep time), milliseconds.
Initially it is OptWaitFor.Period, optionally multiplied by constructor's options.Period/10. Default 10 ms. Then each WaitFor.Loop.Sleep increments it until WaitFor.Loop.MaxPeriod.

Methods
  NameDescription
IsTimeout
If the secondsTimeout time is not expired, returns false.
Else if secondsTimeout is negative, returns true.
Else throws System.TimeoutException.
Sleep
Calls WaitFor.Loop.IsTimeout. If it returns true, returns false.
Else sleeps for WaitFor.Loop.Period milliseconds, increments Period if it is less than WaitFor.Loop.MaxPeriod, and returns true.

Remarks
See examples. The code works like most 'wait for' functions of this library: on timeout throws exception, unless secondsTimeout is negative.
Similar code is used by most 'wait for' functions of this library.
See also WaitFor.Condition; usually it's easier; internally it uses similar code too.
Examples
Copy
public static bool WaitForMouseLeftButtonDown(double secondsTimeout)
{
    var x = new WaitFor.Loop(secondsTimeout);
    for(; ; ) {
        if(Mouse.IsPressed(MButtons.Left)) return true;
        if(!x.Sleep()) return false;
    }
}
The same with WaitFor.Condition.
Copy
static bool WaitForMouseLeftButtonDown2(double secondsTimeout)
{
    return WaitFor.Condition(secondsTimeout, () => Mouse.IsPressed(MButtons.Left));
}
