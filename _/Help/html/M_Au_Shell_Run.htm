ShellRun Method

Runs/opens a program, document, folder, URL, new email, Control Panel item etc.
By default returns process id. If used flag WaitForExit, returns process exit code.
Returns 0 if did not start new process or did not get process handle, usually because opened the document in an existing process.

file

What to run. Can be:
Full path of a file or folder. Examples: @"C:\folder\file.txt", Folders.System + "notepad.exe", @"%Folders.System%\notepad.exe" .
Filename of a file or folder, like "notepad.exe". The function calls SearchPath(String, String) .
Path relative to ThisApp. Examples: "x.exe", @"subfolder\x.exe", @".\subfolder\x.exe", @"..\another folder\x.exe" .
URL. Examples: "http://a.b.c/d", "file:///path" .
Email, like "mailto:a@b.c" . Subject, body etc also can be specified, and Google knows how.
Shell object's ITEMIDLIST like ":: HexEncodedITEMIDLIST". See ToHexString, FoldersVirtual . Can be used to open virtual folders and items like Control Panel.
Shell object's parsing name, like @"::{CLSID}". See ToShellString(NativeSIGDN, Boolean), FoldersVirtualPidl . Can be used to open virtual folders and items like Control Panel.
To run a Windows Store App, use @"shell:AppsFolder\WinStoreAppId" format. Examples: @"shell:AppsFolder\Microsoft.WindowsCalculator_8wekyb3d8bbwe!App", @"shell:AppsFolder\windows.immersivecontrolpanel_cw5n1h2txyewy!microsoft.windows.immersivecontrolpanel". To discover the string use GetWindowsStoreAppId(Wnd, Boolean, Boolean) or Google.
Supports environment variables, like @"%TMP%\file.txt". See ExpandEnvVar(String).

Remarks
It works like when you double-click an icon. It may start new process or not. For example it may just activate window if the program is already running.
Uses API ShellExecuteEx .
Similar to Start(String, String) .
The returned process id can be used to find a window of the new process. Example:
Copy
Wnd w = WaitFor.WindowActive(10, "*- Notepad", "Notepad", Shell.Run("notepad.exe"));
Examples
Copy
Wnd w = Wnd.Find("*- Notepad", "Notepad");
if(w.Is0) { Shell.Run("notepad.exe"); w = WaitFor.WindowActive(Wnd.LastFind); }
w.Activate();
