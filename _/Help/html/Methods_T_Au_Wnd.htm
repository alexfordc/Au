Wnd Methods

The Wnd type exposes the following members.

Methods
  NameDescription
Activate
Activates this window (brings to the foreground).
Also unhides, restores minimized etc, to ensure that the window is ready to receive sent keys, mouse clicks ect.
ActivateLL
Low-level version of Activate .
Just calls EnableActivate, API SetForegroundWindow and makes sure that it actually worked, but does not check whether it activated exactly this window.
No exceptions, does not unhide, does not restore minimized, does not check is it a top-level window or control, etc.
Returns false if fails.
AllChildren
Gets child controls.
Returns array containing 0 or more control handles as Wnd.
ButtonClick(Int32, Boolean)
Finds a child button by id and sends a "click" message. Does not use the mouse.
Calls Click(Boolean).
ButtonClick(String, String, Boolean)
Finds a child button by name and sends a "click" message. Does not use the mouse.
Calls Click(Boolean).
Child
Finds a child control and returns its handle as Wnd.
Returns default(Wnd) if not found. See also: Is0, OrThrow(Wnd), UnaryPlus(Wnd).
ChildAll
Finds all matching child controls.
Returns List containing 0 or more control handles as Wnd.
Everything except the return type is the same as with Child.
ChildFast
Finds a direct child control and returns its handle as Wnd.
Returns default(Wnd) if not found. See also: Is0, OrThrow(Wnd), UnaryPlus(Wnd) .
Calls API FindWindowEx .
Faster than Child, which uses API EnumChildWindows .
Can be used only when you know full name and/or class name.
Finds hidden controls too.
ChildFromXY
Gets child control from point.
Returns default(Wnd) if the point is not in a child control or not in the client area of this window.
ClassNameIs(String)
Returns true if the class name of this window matches className. Else returns false.
Also returns false when fails (probably window closed or 0 handle). Supports GetError.
ClassNameIs(String)
If the class name of this window matches one of strings in classNames, returns 1-based index of the string. Else returns 0.
Also returns 0 if fails to get class name (probably window closed or 0 handle). Supports GetError.
Close
Closes the window.
Returns true if successfuly closed or if it was already closed (the handle is 0 or invalid) or if noWait==true.
ContainsScreenXY
Returns true if this window (its rectangle) contains the specified point in primary screen coordinates.
ContainsWindowXY(Coord, Coord)
This overload calls ContainsWindowXY(WndWindow, x, y).
ContainsWindowXY(Wnd, Coord, Coord)
Returns true if this control (its rectangle) contains the specified point in parent window.
Enable
Enables or disables the window.
Calls API EnableWindow.
EnsureInScreen
Moves this window if need, to ensure that entire window is in screen.
Equals(Object)
Returns true if obj is this Wnd.
(Overrides ValueTypeEquals(Object).)
Equals(Wnd)
Returns true if w == this.
Equals(NullableWnd)
Returns true if w != null and w.Value == this.
Find
Finds a top-level window and returns its handle as Wnd.
Returns default(Wnd) if not found. See also: Is0, OrThrow(Wnd), UnaryPlus(Wnd). See examples.
FindAll
Finds all matching windows.
Returns array containing 0 or more window handles as Wnd.
Everything except the return type is the same as with Find(String, String, String, WFFlags, FuncWnd, Boolean, Object).
FindFast
Finds a top-level window and returns its handle as Wnd.
Returns default(Wnd) if not found. To check it you can use Is0 or OrThrow(Wnd) .
Calls API FindWindowEx .
Faster than Find, which uses API EnumWindows .
Can be used only when you know full name and/or class name.
Finds hidden windows too.
FindOrRun
Finds a top-level window. If found, activates (optionally), else calls callback function that should open the window (eg call Run(String, String, SRFlags, SRMore) ) and waits for the window.
Returns window handle as Wnd. Returns default(Wnd) if not found (if runWaitTimeoutS is negative; else exception).
The first 5 parameters are the same as Find(String, String, String, WFFlags, FuncWnd, Boolean, Object).
Focus
Sets the keyboard input focus to this control or window.
Also activetes its top-level parent window (see Activate ).
Can belong to any process/thread. With controls of this thread you can use the more lightweight function FocusLocal.
FocusLocal
Calls API SetFocus , which sets the keyboard input focus to this control or window, which must be of this thread.
Fails if it belongs to another thread or is invalid or disabled.
Returns false if fails. Supports GetError.
FromMouse
Gets visible top-level window or control from mouse cursor position.
More info: FromXY(Coord, Coord, WXYFlags, CoordOptions).
FromXY(Point, WXYFlags)
Gets visible top-level window or control from point.
More info: FromXY(Coord, Coord, WXYFlags, CoordOptions).
FromXY(Coord, Coord, WXYFlags, CoordOptions)
Gets visible top-level window or control from point.
By default the coordinates are relative to the primary screen.
GetClientRect
Gets client area rectangle.
The same as the ClientRect property.
The same as GetClientSize(Size), just the parameter type is different.
GetClientSize
Gets client area width and height.
The same as the ClientSize property.
The same as GetClientRect(RECT), just the parameter type is different.
GetRect
Gets rectangle (position and size) in screen coordinates.
The same as the Rect property.
GetRectInClientOf
Gets rectangle of this window (usually control) relative to the client area of another window (usually the parent).
GetRectNotMinMax
Gets rectangle of normal (restored) window even if currently it is minimized or maximized.
GetSize
Gets width and height.
The same as the Size property.
GetText
Gets window/control name or control text.
Returns "" if it is empty.
Returns null if fails, eg if the window is closed. Supports GetError .
This is a low-level function. You can instead use Name and ControlText.
GetThreadProcessId
Calls API GetWindowThreadProcessId .
Returns thread id and also gets process id.
Returns 0 if fails. Supports GetError.
 Note
It is native thread id, not Thread.ManagedThreadId.
GetWindowAndClientRectInScreen
Gets window rectangle and client area rectangle, both in screen coordinates.
GetWindowLong
Calls API GetWindowLong if this process is 32-bit, GetWindowLongPtr if 64-bit.
HasAcc
Returns true if this window contains the specified accessible object.
Calls Find(Wnd, WndChildFinder).
HasChild(WndChildFinder)
Returns true if this window contains the specified control.
Calls Find(Wnd).
HasChild(String, String, WCFlags, FuncWnd, Boolean, Int32)
Returns true if this window contains the specified control.
Calls Child(String, String, WCFlags, FuncWnd, Boolean, Int32).
 Note
Using this function many times with same parameters is inefficient. Instead create new WndChildFinder and call Find(Wnd) or HasChild(WndChildFinder). See example.
HasExStyle
Returns true if the window has all specified extended style flags (see ExStyle).
HasStyle
Returns true if the window has all specified style flags (see Style).
IsChildOf
Returns true if this is a direct or indirect child (descendant) of window w.
Calls API IsChild.
IsSpecHwnd
Returns true if w contains a non-zero special handle value (see SpecHwnd ).
Note that HWND_TOP is 0.
Kid
Finds a child control by its id and returns its handle as Wnd.
Returns default(Wnd) if not found. See also: Is0, OrThrow(Wnd), UnaryPlus(Wnd).
MapClientToClientOf(Wnd, RECT)
Converts coordinates relative to the client area of this window to coordinates relative to the client area of window w.
MapClientToClientOf(Wnd, Point)
Converts coordinates relative to the client area of this window to coordinates relative to the client area of window w.
MapClientToScreen(Point)
Converts coordinates relative to the client area of this window to coordinates relative to the screen.
MapClientToScreen(RECT)
Converts coordinates relative to the client area of this window to coordinates relative to the screen.
MapClientToWindow(Point)
Converts coordinates relative to the client area of this window to coordinates relative to the top-left corner of this window.
MapClientToWindow(RECT)
Converts coordinates relative to the client area of this window to coordinates relative to the top-left corner of this window.
MapScreenToClient(Point)
Converts coordinates relative to the screen to coordinates relative to the client area of this window.
MapScreenToClient(RECT)
Converts coordinates relative to the screen to coordinates relative to the client area of this window.
MapWindowToClient(Point)
Converts coordinates relative to the top-left corner of this window to coordinates relative to the client area of this window.
MapWindowToClient(RECT)
Converts coordinates relative to the top-left corner of this window to coordinates relative to the client area of this window.
MapWindowToScreen(Point)
Converts coordinates relative to the top-left corner of this window to screen coordinates.
MapWindowToScreen(RECT)
Converts coordinates relative to the top-left corner of this window to screen coordinates.
MenuClick
Posts a "menu item clicked" notification (WM_COMMAND) as if that menu item was clicked. Does not use the mouse.
Move(Coord, Coord, CoordOptions)
Moves.
Calls Move(x, y, null, null, workArea).
With windows of current thread usually it's better to use MoveLL(Int32, Int32).
Move(Coord, Coord, Coord, Coord, CoordOptions)
Moves and/or resizes.
With windows of current thread usually it's better to use MoveLL(Int32, Int32, Int32, Int32, UInt32).
MoveInScreen
Moves this window to coordinates x y in specified screen, and ensures that entire window is in screen.
MoveLL(Int32, Int32)
Moves.
MoveLL(Int32, Int32, Int32, Int32, UInt32)
Moves and resizes.
MoveToScreenCenter
Moves this window to the center of the screen.
Calls ShowNotMinMax(true) and MoveInScreen(null, null, screen, true, true).
Post
Calls API PostMessage .
Returns its return value (false if failed). Supports GetError.
Resize
Resizes.
Calls Move(null, null, width, height, workArea).
With windows of current thread usually it's better to use ResizeLL(Int32, Int32).
ResizeLL
Resizes.
RestorePositionSizeState
Restores window position, size and state that is stored in a string created by SavePositionSizeState(Boolean).
SavePositionSizeState
Gets window position, size and state stored in a string that can be used with RestorePositionSizeState(String, Boolean, Boolean) .
Returns null if failed. Supports GetError.
Send
Calls API SendMessage.
SendNotify
Calls API SendNotifyMessage .
Returns its return value (false if failed). Supports GetError.
SendS(UInt32, LPARAM, Char)
Calls API SendMessage where lParam is char[].
SendS(UInt32, LPARAM, String)
Calls API SendMessage where lParam is string.
SendTimeout(Int32, UInt32, LPARAM, LPARAM, UInt32)
Calls API SendMessageTimeout .
Returns its return value (false if failed). Supports GetError .
flags can be Native.SMTO_.
SendTimeout(Int32, LPARAM, UInt32, LPARAM, LPARAM, UInt32)
Calls API SendMessageTimeout and gets the result of the message processing.
Returns its return value (false if failed). Supports GetError .
flags can be Native.SMTO_.
SendTimeoutS(Int32, LPARAM, UInt32, LPARAM, Char, UInt32)
Calls API SendMessageTimeout where lParam is char[].
Returns its return value (false if failed). Supports GetError .
flags can be Native.SMTO_.
SendTimeoutS(Int32, LPARAM, UInt32, LPARAM, String, UInt32)
Calls API SendMessageTimeout where lParam is string.
Returns its return value (false if failed). Supports GetError .
flags can be Native.SMTO_.
SetClientSize
Calculates and sets window rectangle from the specified client area rectangle.
Calls ResizeLL.
SetExStyle
Changes window extended style.
SetStyle
Changes window style.
SetText
Sets window/control name or control text.
SetTransparency
Sets transparency.
On Windows 7 works only with top-level windows, on newer OS also with controls.
SetWindowLong
Calls API SetWindowLong if this process is 32-bit, SetWindowLongPtr if 64-bit.
SetWindowPos
Calls API SetWindowPos.
Show
Shows (if hidden) or hides this window.
Does not activate/deactivate/zorder.
With windows of current thread usually it's better to use ShowLL.
ShowLL
Shows (if hidden) or hides this window.
Does not activate/deactivate/zorder.
ShowMaximized
If not minimized, minimizes.
Also unhides.
ShowMinimized
If not minimized, minimizes.
Also unhides.
ShowNotMinimized
If minimized, restores previous non-minimized state (maximized or normal).
Also unhides.
ShowNotMinMax
If maximized or minimized, makes normal (not min/max).
Also unhides.
ThrowIf0
If Is0, throws WndException.
ThrowIfInvalid
If Is0 or !IsAlive, throws WndException.
ThrowNoNative
Throws WndException that uses mainMessage and does not use the last Windows API error.
Also the message depends on whether the window handle is 0/invalid.
ThrowUseNative
Throws WndException that uses the last Windows API error (code and message).
Also the message depends on whether the window handle is 0/invalid.
ThrowUseNative(String)
Throws WndException that uses mainMessage and the last Windows API error (code and message).
Also the message depends on whether the window handle is 0/invalid.
ThrowUseNative(Int32, String)
Throws WndException that uses mainMessage and the specified Windows API error code.
Also the message depends on whether the window handle is 0/invalid.
ToString
Formats string $"{handle} {ClassName} \"{Name}\" {ProcessName} {Rect}".
(Overrides ValueTypeToString.)
WndChild
Gets the child control at the specified position in the Z order.
Returns default(Wnd) if no children or if index is invalid.
ZorderAbove
Places this window before window w in the Z order.
Also can make this window topmost or non-topmost, depending on where w is in the Z order.
This window and w can be both top-level windows or both controls of same parent.
May not work with top-level windows when it would move an inactive window above the active window.
If w is default(Wnd), calls ZorderBottom.
ZorderBelow
Places this window after window w in the Z order.
Also can make this window topmost or non-topmost, depending on where w is in the Z order.
This window and w can be both top-level windows or both controls of same parent.
May not work with top-level windows when it would move an inactive window above the active window.
If w is default(Wnd), calls ZorderTop.
ZorderBottom
Places this window or control at the bottom of the Z order.
If the window was topmost, makes it and its owner window non-topmost.
ZorderIsBefore
Returns true if this window is above window w in the Z order.
ZorderNoTopmost
Makes this window non-topmost.
If this window has an owner window, makes the owner window non-topmost too.
This cannot be a control.
ZorderTop
Places this window or control at the top of the Z order.
If the window was topmost, it will be at the top of topmost windows, else at the top of non-topmost windows (after topmost windows).
Does not activate.
In most cases does not work with top-level inactive windows, although returns true; instead use ActivateLL.
ZorderTopmost
Makes this window topmost (always on top of non-topmost windows in the Z order).
Does not activate.
If this window has an owner window, the owner does not become topmost.
This cannot be a control.

By default x y coordinates are relative to the client area; default - center.
