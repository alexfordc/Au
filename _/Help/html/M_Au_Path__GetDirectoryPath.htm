Path_GetDirectoryPath Method

Removes filename part from path. By default also removes separator ('\\' or '/') if it is not after drive name (eg "C:").
Returns "" if the string is a filename.
Returns null if the string is null or a root (like @"C:\" or "C:" or @"\\server\share" or "http:").

Don't remove the separator character(s) ('\\' or '/').
Examples: from @"C:\A\B" gets @"C:\A\", not @"C:\A"; from "http://x.y" gets "http://", not "http:".

Return Value

Remarks
Similar to System.IO.Path.GetDirectoryName. Some diferences: does not throw exceptions; skips '\\' or '/' at the end (eg from @"C:\A\B\" gets @"C:\A", not @"C:\A\B"); does not expand DOS path; much faster.
Parses raw string. You may want to Path_.Normalize it at first.
Supports separators '\\' and '/'.
Also supports URL and shell parsing names like @"::{CLSID-1}\0\::{CLSID-2}".
Example paths and results (withSeparator=false):
Copy
@"C:\A\B\file.txt" -> @"C:\A\B".
"file.txt" -> "".
@"C:\A\B\" -> @"C:\A".
@"C:\A\/B\/" -> @"C:\A".
@"C:\" -> null.
@"\\network\share" -> null.
"http:" -> null.
@"C:\aa\file.txt:alt.stream" -> "C:\aa".
"http://a.b.c" -> "http:".
"::{A}\::{B}" -> "::{A}".
"" -> "".
null -> null.
