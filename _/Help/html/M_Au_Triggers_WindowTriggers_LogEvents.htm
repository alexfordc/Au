Starts or stops to print window events that can help to create or debug window triggers.

Parameters

on
Type: SystemBoolean
Start (true) or stop.
skip (Optional)
Type: SystemFuncWnd, Boolean
An optional callback function that can be used to reduce noise, eg skip tooltip windows. Return true to skip that window.
Remarks
For primary trigger events is logged this info:
  1. Time milliseconds. Shows only the remainder of dividing by 10 seconds, therefore it starts from 0 again when reached 9999 (9 seconds and 999 milliseconds).
  2. Event (see Triggers.TWEvents).
  3. Letters for window state etc:
    • A - the window is active.
    • H - the window is invisible (!Wnd.IsVisible).
    • C - the window is cloaked (Wnd.IsCloaked).
    • O - the window is considered old, ie created before calling Triggers.Run.
    • T - the even has been detected using a timer, which means slower response time. Else detected using a hook.
  4. Window (handle, class, name, program, rectangle).
Colors are used for window event types used for primary triggers: blue if activated; green if became visible; yellow if name changed.
For "later" events is logged time, event and window. Black, tab-indented. Prints only events that are specified in triggers.
When a trigger is activated, prints the event type in red.
Examples

Triggers.Window.LogEvents(true, o => 0 != o.ClassNameIs("*tooltip*", "SysShadow", "#32774", "TaskList*"));
