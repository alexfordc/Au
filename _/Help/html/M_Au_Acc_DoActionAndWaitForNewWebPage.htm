AccDoActionAndWaitForNewWebPage Method

Calls Acc.DoAction or action and waits until window name changes and web page name changes.

Namespace:  Au
Assembly:  Au (in Au.dll) Version: 1.0.0.0

public bool DoActionAndWaitForNewWebPage(
	double secondsTimeout = 60,
	Action<Acc> action = null
)

Parameters

secondsTimeout (Optional)
Type: SystemDouble
The maximal time to wait, seconds. If 0, waits infinitely. If >0, after that time interval throws System.TimeoutException. If <0, after that time interval returns false.
Default 60 seconds.
action (Optional)
Type: SystemActionAcc
If used, calls it instead of Acc.DoAction.

Return Value

Type: Boolean
Returns true. On timeout returns false if secondsTimeout is negative; else exception.
Exceptions
ExceptionCondition
TimeoutExceptionsecondsTimeout time has expired (if > 0).
AuExceptionFailed. For example, when this object is invalid, or its top-level window does not contain a web page.
WndExceptionThe window was closed while waiting.
ExceptionExceptions thrown by Acc.DoAction or by the action function.
Remarks
This function is used to click a link in a web page and wait until current web page is gone. It prevents a following 'wait for object' function from finding a matching object in the old page, which would be bad.
This function does not wait until the new page is completely loaded. There is no reliable/universal way for it. Instead, after calling it you can call a 'wait for object' function which waits for a known object that must be in the new page.
This function cannot be used when the new page has the same title as current page. Then it waits until secondsTimeout time or forever. The same if the action does not open a web page.
