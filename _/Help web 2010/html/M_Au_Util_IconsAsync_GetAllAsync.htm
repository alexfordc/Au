Remarks
After this function returns, icons are asynchronously extracted with Icon_.GetFileIconHandle, and callback called with icon handle (or default(IntPtr) if failed).
The callback is called in this thread. This thread must have a message loop (eg Application.Run).
If you'll need more icons, you can add more files and call this function again with the same IconsAsync instance, even if getting old icons if still not finished.
