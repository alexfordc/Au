Parameters

name (Optional)
Type: SystemString
Control name.
String format: wildcard expression.
null means 'can be any'. "" means 'must not have name'.

By default the function gets control names with Wnd.Name.
Can start with these prefix strings:
  • "***text " - use Wnd.ControlText.
    Slower and can be less reliable (because can get editable text), especially if not used cn (class name). Does not remove the invisible '&' characters that are used to underline keyboard shortcuts with the Alt key.
  • "***accName " - use Wnd.NameAcc.
    Useful when the control itself does not have a name but an adjacent Static text control is used as its name. Examples - Edit controls in dialogs. Slower.
  • "***wfName " - use .NET Windows Forms Control Name property.
    To get it this function uses Wnd.Misc.WinFormsControlNames. It is slower and can fail because of UAC.
  • "***id " (like "***id 15") - use control id.
    To get it this function uses Wnd.ControlId.
    The id value cannot be wildcard expression.
    See also Wnd.ChildById.
cn (Optional)
Type: SystemString
Control class name.
String format: wildcard expression.
null means 'can be any'. Cannot be "".
You can see control class name etc in editor's status bar and dialog "Find window or control".
flags (Optional)

also (Optional)

Callback function. Called for each matching control.
It can evaluate more properties of the control and return true when they match.
Example: also: t => t.IsEnabled
skip (Optional)
Type: SystemInt32
0-based index of matching control.
For example, if 1, the function skips the first matching control and returns the second.
