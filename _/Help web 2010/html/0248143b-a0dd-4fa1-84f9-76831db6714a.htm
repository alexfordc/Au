Wildcard expression

Au Class Library

Wildcard expression - a simple text format that supports wildcard characters, regular expression, multiple parts and 'must not match'. Like a regular expression, but much simpler. Can be used with 'find' functions, for example Wnd.Find.

Wildcard expression reference

By default case-insensitive. Always culture-insensitive.

By default, if contains wildcard characters, is compared as wildcard pattern (see Like_), else as simple text (whole, not partial).

Wildcard characters

*

Zero or more of any characters.

Examples: "start*", "*end", "*middle*".

?

Any character.

Example: "date ????-??-??"

 Note

There are no escape sequences for * and ? characters. To match literal * or ? character, usually can be used the ? wildcard character. Or you can use regular expression.

Can start with "**options ", like "**tc text".

Options

t

Simple text, not wildcard.

Example: "**t text"

r

PCRE regular expression. See Regex_. Syntax: full, short.

R

.NET regular expression. See Regex. Cannot be used with Acc class functions.

c

Case-sensitive.

Example: "**rc regex"

n

Must not match.

m

Multi-part.

Example: "**m findThis||orThis||**r orThisRegex||**n butNotThis||**nr andNotThisRegex"

Example 2: "**mn notThis||andNotThis"

Default separator is ||. Custom separator example: "**m(^^^) findThis^^^orThis".

 Note

Exception ArgumentException if invalid "**options " or regular expression.

Wildcard expression "" matches only "".

If the parameter is or omitted, it usually means 'match any'.

 Tip

To avoid many nulls in code, you can omit optional parameters, for example instaed of Wnd.Find(null, null, "notepad") use Wnd.Find(programEtc: "notepad") .

Example
Copy
//Find item whose property 1 is "example" (case-insensitive), property 2 starts with "2017-" and property 3 matches a case-sensitive regular expression.
var item = x.FindItem("example", "2017-*", "**rc regex");
